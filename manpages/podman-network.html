    NAME

       podman-network - Manage Podman networks

    SYNOPSIS

       podman network subcommand

    DESCRIPTION

       The network command manages networks for Podman.

   Podman  supports  two network backends Netavark and CNI. Support for netavark was added in Podman v4.0. To configure the network backend use the network_backendkey under the [Net‐
   work] in containers.conf(5).  New systems should use netavark by default, to check what backed is used run podman info --format {{.Host.NetworkBackend}}.

   All network commands work for both backends but CNI and Netavark use different config files so networks have to be created again after a backend change.
